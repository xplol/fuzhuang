# 部署说明

本文档提供通用部署方式，不包含任何服务器密码、面板地址或客户私有信息。

## 1. 环境要求

- Python 3.9+
- Nginx
- systemd
- 可写的网站目录

## 2. 后端启动

在项目目录执行：

```bash
python3 api_server.py
```

默认监听：

```text
127.0.0.1:8301
```

数据库会自动创建在：

```text
data/jiangsu_fuzhuang.sqlite3
```

## 3. systemd 服务示例

将下面内容保存为 `/etc/systemd/system/fuzhuang-api.service`，路径按实际项目位置调整。

```ini
[Unit]
Description=Fuzhuang Management API
After=network.target

[Service]
Type=simple
WorkingDirectory=/www/wwwroot/fuzhuang
ExecStart=/usr/bin/python3 /www/wwwroot/fuzhuang/api_server.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
systemctl daemon-reload
systemctl enable --now fuzhuang-api.service
systemctl status fuzhuang-api.service
```

## 4. Nginx 配置示例

```nginx
server {
    listen 80;
    server_name _;

    root /www/wwwroot/fuzhuang;
    index index.html;

    location /api/ {
        proxy_pass http://127.0.0.1:8301/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

检查并重载：

```bash
nginx -t
systemctl reload nginx
```

## 5. 宝塔部署提示

如果使用宝塔：

1. 新建一个静态站点。
2. 上传项目文件到站点目录。
3. 添加 Nginx 反向代理规则，将 `/api/` 代理到 `127.0.0.1:8301`。
4. 使用 systemd 或宝塔进程守护启动 `api_server.py`。

系统使用 SQLite 文件数据库，因此不会出现在宝塔 MySQL 数据库列表中。

## 6. 备份建议

重点备份：

```text
data/jiangsu_fuzhuang.sqlite3
assets/
templates/
```

更新代码前建议先备份整个项目目录，尤其是 `data/` 目录。

## 7. 上线后必须修改

- 默认管理员密码
- 默认二次密码
- Nginx 站点域名或访问策略
- 数据库备份策略
